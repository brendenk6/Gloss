# Gloss

Two-LLM collaborative drawing app. Claude and Codex both draw on a shared CGContext-backed canvas via HTTP API. Brenden watches in real-time.

## Architecture

```
Gloss/
├── project.yml                   ← xcodegen source of truth
├── Gloss.xcodeproj               ← REGENERATED by xcodegen, don't hand-edit
└── Gloss/
    ├── GlossApp.swift            ← App entry, single window
    ├── Info.plist
    ├── Gloss.entitlements
    ├── Models/
    │   ├── CanvasStore.swift     ← CGContext source of truth, MainActor command processor, revision counter
    │   ├── DrawCommand.swift     ← Codable command types (stroke/shape/text/image/clear/undo/redo)
```

## Source of truth

`CanvasStore` owns a single `CGContext` of size W×H, sRGB premultiplied RGBA. CGContext is **not thread-safe** — every mutation goes through `apply(_ command: DrawCommand)` on `@MainActor`. Snapshots return cached `CGImage`s, refreshed on revision change.

```swift
public protocol CanvasStoreContract {
    @MainActor func apply(_ command: DrawCommand) async throws -> CommandResult
    @MainActor func snapshot(maxDim: Int?) -> CGImage
    @MainActor func snapshot(region: CGRect, scale: CGFloat) -> CGImage
    @MainActor func sample(at: CGPoint) -> RGBA?
    @MainActor var state: CanvasState { get }   // width, height, revision, lastCommands, authorCursors
}
```

`CommandResult` carries `{ revision, dirtyRect }`. The HTTP layer returns this in JSON responses.

## HTTP API (port :7778, localhost-only)

| Method | Route | Purpose |
|--------|-------|---------|
| GET | /state | width, height, revision, lastCommands[recent], authorCursors |
| GET | /canvas.png?max_dim=N | Full canvas, optional downscale |
| GET | /region.png?x&y&w&h&scale= | Region of canvas at native or scaled resolution |
| GET | /sample?x&y | Single-pixel RGBA value (eyedropper) |
| POST | /stroke | author, points[], width, color, opacity, blend, idempotencyKey |
| POST | /shape | rect/ellipse/line + stroke + fill + author + idempotencyKey |
| POST | /text | string, point, font, size, color, author |
| POST | /image | base64 PNG, point, blend, author |
| POST | /clear | clear canvas (irreversible unless undo enabled) |
| POST | /undo | rewind one command |
| POST | /redo | replay one command |
| POST | /resize | new width/height (clears or stretches per option) |

All POSTs return `{ ok, revision, dirtyRect }`. Errors return `{ ok: false, error: { code, message } }`.

## Coordinates

Top-left origin, y down. Matches Apple imaging convention. Pixel-precise (floats accepted, rasterized at draw time).

## Idempotency

All mutation commands accept optional `idempotencyKey: String`. The server keeps a small recent-key cache (~256 entries) and silently no-ops on repeat. The CanvasStore also dedupes by key as defense in depth — retries from Claude/Codex/Brenden never double-draw.

## Multi-LLM coordination

ONE shared canvas, attribution per command. Each command carries `author: String` (convention: "claude" / "codex" / "brenden"). Commands stomp each other freely — collaboration emerges from us watching the canvas and reacting. Per-LLM layers is a v2 feature; the data model already carries an optional `layerID` so layers are additive.

## Build workflow

```bash
cd ~/Projects/Gloss
xcodegen
xcodebuild -project Gloss.xcodeproj -scheme Gloss -configuration Debug \
  -destination 'platform=macOS' build

# Build output:
# /tmp/XcodeDerivedData/Gloss-*/Build/Products/Debug/Gloss.app

# Run:
pkill -x Gloss; sleep 1
open /tmp/XcodeDerivedData/Gloss-*/Build/Products/Debug/Gloss.app
```

For a built `/Applications/Gloss.app` install:

```bash
xcodebuild -project Gloss.xcodeproj -scheme Gloss -configuration Release build
pkill -x Gloss; sleep 2
/usr/bin/ditto /tmp/XcodeDerivedData/Gloss-*/Build/Products/Release/Gloss.app /Applications/Gloss.app
open /Applications/Gloss.app
```

## Aesthetic

Pearl / lavender / aqua / glow palette. Liquid Glass on macOS 26 chrome. The canvas itself is plain white by default (don't tint the working surface — it'd skew color picking).
